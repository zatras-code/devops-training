Расчёт ресурсов для Kubernetes-кластера
Исходные данные
| Компонент   | RAM на 1 копию | CPU на 1 копию | Кол-во копий |      Всего RAM | Всего CPU |
| ----------- | -------------: | -------------: | -----------: | -------------: | --------: |
| База данных |           4 ГБ |          1 CPU |            3 |          12 ГБ |     3 CPU |
| Кеш         |           4 ГБ |          1 CPU |            3 |          12 ГБ |     3 CPU |
| Frontend    |          50 МБ |        0.2 CPU |            5 |         250 МБ |     1 CPU |
| Backend     |         600 МБ |          1 CPU |           10 |           6 ГБ |    10 CPU |
Итого для приложения требуется:
CPU: 3 + 3 + 1 + 10 = 17 CPU
RAM: 12 + 12 + 0.25 + 6 = 30.25 ГБ (округлим RAM до 31 ГБ)

Запас на отказ одной рабочей ноды.
Чтобы кластер продолжал работать при выходе из строя одной worker-ноды, ресурсов должно хватать даже после потери одной ноды.

Возьмём worker-ноды:
4 worker-ноды каждая: 8 CPU, 16 ГБ RAM

Общие ресурсы worker-нод:

CPU: 4 × 8 = 32 CPU
RAM: 4 × 16 = 64 ГБ

С учётом служебных ресурсов на каждой worker-ноде зарезервируем примерно: 0.5 CPU и 1 ГБ RAM

Полезные ресурсы одной worker-ноды: 7.5 CPU и 15 ГБ RAM
Полезные ресурсы 4 worker-нод: CPU: 4 × 7.5 = 30 CPU, RAM: 4 × 15 = 60 ГБ
После выхода из строя одной worker-ноды останется 3 worker-ноды: CPU: 3 × 7.5 = 22.5 CPU, RAM: 3 × 15 = 45 ГБ
Этого достаточно для нагрузки приложения:
требуется 17 CPU, доступно 22.5 CPU
требуется 31 ГБ RAM, доступно 45 ГБ RAM

Control-plane ноды
Для отказоустойчивости control-plane желательно использовать 3 ноды.
Параметры control-plane-ноды: 2 CPU, 4 ГБ RAM
Итого control-plane 3 ноды: 6 CPU, 12 ГБ RAM

Итоговая конфигурация кластера
| Тип ноды      | Кол-во | CPU на ноду | RAM на ноду | Итого CPU | Итого RAM |
| ------------- | -----: | ----------: | ----------: | --------: | --------: |
| Control-plane |      3 |       2 CPU |        4 ГБ |     6 CPU |     12 ГБ |
| Worker        |      4 |       8 CPU |       16 ГБ |    32 CPU |     64 ГБ |

Итого для проекта требуется Kubernetes-кластер:
3 control-plane-ноды по 2 CPU и 4 ГБ RAM
4 worker-ноды по 8 CPU и 16 ГБ RAM
